LESSON ／ 失敗案先講，免得你重踩

設計稿不能只給人看

鶴步牙醫是這套流程踢到鐵板的真實案。問題不在 MCP、不在 Claude，在 Figma 設計稿本身。

出了什麼事

素材用 Photoshop 風格的疊加模式（multiply / screen 之類）讓白底變透明， 在 Figma 看視覺很完整。

但 AI 把該圖層 export 成一般 PNG／JPG 給 WP 使用後， 疊加效果消失、白底回來了，整個視覺崩掉，跟原稿差很多。

TAKEAWAY

設計階段就要想「AI 會怎麼下載這張圖」

  • 透明效果做進素材本身，不靠上層 blend mode
  • 每個圖層獨立 export 出來都要是「對的視覺」
  • Mask、特效、調整圖層上線前先 flatten 確認
  • 圖層命名、frame 分組要讓 AI 找得到、抓得到

✦ 結論：Figma 不只是給設計師看，它是 AI 切版的原料。原料有問題，後面 pipeline 再強都救不回來。

PITFALLS ／ 連線部署類 (1/2)

前 30 分鐘最常踩的雷

PITFALL 01

Application Password 401 Unauthorized ／ 複製時空格被吃掉

解：到 WP Profile 重新產一組 → 整組含空格貼到 settings.json，存檔後完全關掉 Claude Code 再重開。

PITFALL 02

部署成功但前台頁面沒變 ／ Elementor 快取沒重建

解 3 選 1：URL 加 ?fresh=<時間戳>、WP Admin → Elementor Tools → Regenerate CSS、或部署成 draft 再切 publish。

PITFALL 03

widget 上設的 css_classes 沒 render 出來 ／ 只有 container 才會輸出

解：CSS selector 改用 .elementor-element-{widget_id}，widget 的 ID 一定會變成 class。

PITFALL 04

MCP 部署回 true 但前台沒變 ／ 第一次成功、第二次起 silent fail（線上站特別容易）

解：(1) 換一個全新乾淨頁重部署（每個 trainee 一頁）；(2) fallback 走 WP REST POST /wp/v2/pages/{id} 帶完整 meta payload（_elementor_data, _elementor_version, _wp_page_template）。

PITFALLS ／ 切版排版類 (2/2)

理仁牙醫切版時踩出來的雷

PITFALL 04

e-con 容器內建 padding + max-width 1140 ／ 全寬背景永遠縮成 1140 寬

解：section 加 reset：--padding-*:0、--content-width:100%、max-width:100% !important，再用 inner 1440 包置中內容。

PITFALL 05

絕對定位跑版／圖片跳到下段 ／ 父容器沒 position:relative

解：用 HTML widget 自己寫 wrapper 強制 relative。對抗 Elementor 超過 3 個 !important 就停手改 HTML widget，最乾淨。

ASSETS ／ 圖資處理 SOP

SVG icon 怎麼從 Figma 上 WP

純線稿、單色 vector 用 SVG 最理想（檔小、可縮放、可改色）。流程五步：

# 1. WP 先裝 Safe SVG plugin（沒裝 upload 會擋）
curl -u "$WP_USER:$WP_PASS" -X POST "$WP_URL/wp-json/wp/v2/plugins" \
  -H "Content-Type: application/json" \
  -d '{"slug":"safe-svg","status":"active"}'

# 2. 從 Figma export 出 SVG（要找 instance 內的 icon 子節點，不是整張卡）
// figma MCP use_figma 跑：
const card = await figma.getNodeByIdAsync('1:690');
const circle = card.children.find(c =>
  Math.round(c.width)===180 && Math.round(c.height)===180);
const bytes = await circle.exportAsync({format:'SVG'});
return { b64: figma.base64Encode(bytes) };

# 3. macOS base64 解碼（旗標不能省，pipe 會炸）
base64 -d -i icon.b64 -o icon.svg

# 4. 本機驗證三件套
xmllint --noout icon.svg && grep -c "<path" icon.svg && wc -c icon.svg

# 5. 上傳：用 Python urllib，不要 curl + python pipe（控制字元會炸 json.loads）
python3 -c "import json,base64,urllib.request; ..." # 用 strict=False 解析

✦ 整段 SOP 在 obsidian sop/figma-to-elementor.md §5，含 3 個 macOS 雷跟 instance icon ID 規則。

DEPLOY ／ 線上站三大坑

為什麼部署回 true 卻沒生效

PITFALL 06

JSON 雙重編碼 ／ download_page_to_file 下來的檔案開頭是 "[\"id\"...]"（雙引號 = 已被字串化一層）

直接改完丟回去 → MCP 又 wrap 一層 → 三重編碼 → Elementor 讀不到、fallback 舊版且不報錯。寫回前一定先 json.loads 兩次脫到 list，json.dumps 一次寫回。檔頭必須是 [。

PITFALL 07

Cache 鎖死後續部署 ／ 第一次成功，第二次起 silent fail

本機站有 SQL 直接 DELETE FROM wp_postmeta 三個 meta（_elementor_element_cache、_elementor_page_assets、_elementor_css）。線上站沒 SQL → 換一頁全新部署 是最快路徑。

PITFALL 08

WP 主題 header 蓋住 Hero ／ 頁面上方多出 site title / page title

解：page template 設成 elementor_canvas。REST payload 同時設 template: 'elementor_canvas' 跟 meta._wp_page_template: 'elementor_canvas'，兩個都要。

DEPLOY ／ MCP 失敗時走這個

完整 REST 部署 payload（複製可用）

當 elementor MCP 回 true 但實際沒寫，直接走 WP REST。Python urllib 比 curl 穩（避控制字元）：

import json, base64, urllib.request

USER='wp_admin'; PASS='xxxx xxxx xxxx xxxx xxxx xxxx'
SITE='https://your-wp.example.com'; PAGE_ID=21

auth = base64.b64encode(f'{USER}:{PASS}'.encode()).decode()
elementor_data = json.dumps([
  # your container array here
], ensure_ascii=False)

payload = json.dumps({
  'title': '頁面標題',
  'status': 'publish',
  'template': 'elementor_canvas',
  'meta': {
    '_elementor_data': elementor_data,           # 字串，不是 array
    '_elementor_edit_mode': 'builder',
    '_elementor_template_type': 'wp-page',
    '_elementor_version': '3.30.0',
    '_wp_page_template': 'elementor_canvas',
  }
}, ensure_ascii=False).encode('utf-8')

req = urllib.request.Request(
  f'{SITE}/wp-json/wp/v2/pages/{PAGE_ID}',
  data=payload, method='POST',
  headers={'Authorization': f'Basic {auth}',
           'Content-Type': 'application/json'})
with urllib.request.urlopen(req, timeout=30) as r:
  d = json.loads(r.read().decode('utf-8', errors='replace'), strict=False)
print('modified:', d['modified'], 'template:', d['template'])
